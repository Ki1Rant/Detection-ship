# Ship Detection Pipeline

Полный цикл разработки системы детекции кораблей: от подготовки датасета через кросс-валидацию, дистилляцию знаний и дообучение до инференса на видео с трекингом и SAHI.

## Этапы обработки

### 1. Подготовка датасета (`src/create_folds.py`)
- Разбиение исходного датасета (4885 изображений) на фолды
- Автоматическая верификация пар изображение-метка
- Стратифицированное разделение с фиксированным seed для воспроизводимости

### 2. Кросс-валидационное обучение (`main.py` + `src/src_train/`)
- Обучение ансамбля моделей YOLOv8m и YOLO11m на каждом из 3 фолдов
- Гибкая конфигурация гиперпараметров через `src_train/config.py`

### 3. Дистилляция знаний (`src/create_dist_dataset.py`)
- Прогон неразмеченных данных через ансамбль из 6 моделей (2 архитектуры × 3 фолда)
- **Weighted Boxes Fusion (WBF):** объединение предсказаний с параметрами:
  - Пороги уверенности: 0.10 (для входа в WBF) → 0.30 (финальный)
  - IoU-пороги: 0.35 для объединения
- Генерация высокоточной псевдо-разметки (pseudo-labels)

### 4. Дообучение ансамбля (`finetune_train.py`)
- Специализированное дообучение на 
- Увеличение разрешения до 1024×1024
- Расширенная аугментация: mosaic, масштабирование, erasing
- Обучение на 50 эпох с ранней остановкой 

### 5. Обучение модели-ученика (`main_student_train.py`)
- Обучение легкой модели YOLOv8n на дистиллированном датасете + исходных данных
- Разрешение: 640×640, батч: 32, эпох: 80
- Агрессивная аугментация включая erasing (0.4) для робастности
- Валидация только на Ground Truth разметке

### 6. Инференс с трекингом (`detector.py` + `process_video.py`)
- **ShipDetector:** ансамблевая инференса из дообученных моделей (1024×1024)
- **SAHI (Sliced Aided Hyper Inference):**
  - Разрезание кадров на сегменты 640×640 с перекрытием 25%
  - Детекция на каждом сегменте и постобработка
- **Трекинг объектов:** IoU-based ассоциация с параметрами:
  - Минимум попадания: 3 фрейма
  - Максимальный возраст: 12 фреймов
  - Сглаживание координат: α=0.3
- **Фильтрация:** удаление вложенных боксов (мачты, надстройки)
- Батчевая обработка видео с визуализацией и FPS-оптимизацией

##  Утилиты
- `src/image_viewer.py` — интерактивная просмотра и ручная очистка датасета (Tkinter)
- `src/deduplicate_dataset.py` — удаление дубликатов из датасета
- `src/random_reduce_images.py` — случайное уменьшение изображений

##  Структура выходных данных
- `outputs/checkpoints/` — контрольные точки кросс-валидации
- `outputs/adv_checkpoints/` — дообученные модели для ансамбля
- `outputs/finetune/` — дообученные модели с разрешением 1024×1024
- `outputs/student_distilled/` — обученная модель-ученик
- `fold_dataset/` — структурированный датасет с фолдами и тестовой выборкой
