# Ship Detection Pipeline

Этот репозиторий содержит полный цикл разработки системы детекции кораблей: от подготовки данных и кросс-валидации до дистилляции знаний и инференса на видео.

## Основные компоненты

### 1. Подготовка данных (`create_folds.py`)
*   Разбивает исходный датасет на тестовую выборку и $K$ фолдов для кросс-валидации.
*   Обеспечивает строгое разделение данных.

### 2. Обучение и валидация (`main.py`, `src/`)
*   **Архитектура:** Использование YOLOv8/YOLO11 для обучения на нескольких фолдах.
*   **Конфигурация:** Гибкое управление гиперпараметрами через `config.py`.
*   **Автоматизация:** Полный цикл обучения с автоматическим созданием конфигурационных файлов `data.yaml` для каждого фолда.

### 3. Дистилляция знаний (`create_dist_dataset.py`)
*   **Ансамблирование:** Прогон неразмеченных данных через ансамбль предобученных моделей (YOLOv8m/yolo11m).
*   **WBF (Weighted Boxes Fusion):** Объединение предсказаний ансамбля для получения высокоточных псевдо-меток (pseudo-labels).
*   **Очистка:** Создание "дистиллированного" датасета для обучения компактной модели-ученика (`yolov8n`).

### 4. Обучение Ученика (`main_student_train.py`)
*   Специализированный скрипт для обучения легкой модели (`yolov8n`) на дистиллированном датасете + исходном датасете для минимизации задержки при инференсе.

### 5. Инференс и визуализация (`detector.py`, `process_video.py`)
*   **ShipDetector:** Класс для инференса с поддержкой SAHI для обнаружения мелких объектов на больших кадрах.
*   **Обработка:** Поддержка батчевой обработки видео с отрисовкой боксов и FPS-оптимизацией.
*   **Инструментарий:** `image_viewer.py` — утилита на Tkinter для визуальной проверки и ручной очистки датасета.

## Использование

1.  **Подготовка:** Запустите `create_folds.py` для формирования структуры папок `fold_dataset`.
2.  **Обучение ансамбля:** Используйте `main.py` для получения базовых моделей.
3.  **Дистилляция:** Используйте `create_dist_dataset.py` для генерации псевдо-разметки на новых данных.
4.  **Обучение ученика:** Запустите `main_student_train.py` для получения финальной легкой модели.
5.  **Инференс:** Настройте пути в `process_video.py` и запустите обработку видео.
